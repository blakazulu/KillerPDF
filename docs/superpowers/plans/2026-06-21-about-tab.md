# About Tab + Enriched About Overlay Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a discoverable "About" button to the mode-tab strip that opens the existing About overlay, enrich that overlay with author/repository/license rows, and fix three dead `scalpel.example.com` links.

**Architecture:** Pure WPF UI change in the `MainWindow` monolith plus six localization resource dictionaries. The About overlay (`AboutOverlay`) and its populator (`ShowAboutOverlay()`) already exist; this plan adds an entry point, three info rows, and repoints dead links. One new `Button`-targeted style mirrors the existing mode-tab look without a "checked" state. A new xUnit test guards locale-key parity for the added keys.

**Tech Stack:** .NET Framework 4.8 (net48, x64), WPF, xUnit. Build with the .NET 8+ SDK (`dotnet` or `~/.dotnet/dotnet.exe`).

## Global Constraints

- Build requires the .NET 8 SDK even though the target is `net48`. If `dotnet` is not on PATH, use `~/.dotnet/dotnet.exe`.
- **Localization rule:** every `x:Key` must exist in **all six** `Strings/*.xaml` files (en-US, es, zh-TW, zh-CN, bn, tr-TR), or a `DynamicResource` lookup blanks out in that language.
- Info-card row **labels** are localized and written **UPPERCASE** to match existing labels (`VERSION`, `PUBLISHER`). Proper nouns / URLs (`Liraz Amir`, the GitHub URL, `GPLv3`) are **not** translated.
- The canonical author/repo/license values, copied verbatim:
  - Author: `Liraz Amir`
  - Repository URL: `https://github.com/blakazulu/ScalpelPDF` (display text `github.com/blakazulu/ScalpelPDF`)
  - License URL: `https://www.gnu.org/licenses/gpl-3.0.html` (display text `GPLv3`)
- The About button is a plain `Button`, **never** a grouped `RadioButton` — it must not participate in `AppMode` selection.
- C# style: `Nullable` + `ImplicitUsings` enabled, `LangVersion=latest`; match existing expression-bodied-member / collection-expression idioms.

```csharp
using System.IO;
using System.Linq;
using Xunit;

namespace Scalpel.Tests
{
    public class LocaleParityTests
    {
        private static readonly string[] Locales =
            ["en-US", "es", "zh-TW", "zh-CN", "bn", "tr-TR"];

        private static readonly string[] NewKeys =
            ["Str_Mode_About", "Str_About_Author", "Str_About_Repository", "Str_About_License"];

        // Walk up from the test bin dir to the repo root (the folder that has a Strings dir).
        private static string StringsDir()
        {
            var dir = new DirectoryInfo(System.AppContext.BaseDirectory);
            while (dir != null && !Directory.Exists(Path.Combine(dir.FullName, "Strings")))
                dir = dir.Parent;
            Assert.NotNull(dir);
            return Path.Combine(dir!.FullName, "Strings");
        }

        [Theory]
        [InlineData("Str_Mode_About")]
        [InlineData("Str_About_Author")]
        [InlineData("Str_About_Repository")]
        [InlineData("Str_About_License")]
        public void New_about_keys_exist_in_every_locale(string key)
        {
            var marker = $"x:Key=\"{key}\"";
            foreach (var loc in Locales)
            {
                var text = File.ReadAllText(Path.Combine(StringsDir(), $"{loc}.xaml"));
                Assert.True(text.Contains(marker), $"{loc}.xaml is missing {key}");
            }
        }
    }
}
```
